# 按应用捕获：兼容性与排查

## 兼容性风险

- **进程环回需 Windows 10 21H2+ / Windows 11**：按应用捕获（不静音其他软件）依赖系统进程环回 API。若本机不支持或辅助工具未实现，程序会自动回退为「收录全部系统声音」的设备环回，并在日志中说明。
- **部分应用可能无法被会话级捕获**：使用独占模式、硬件直通或特殊驱动的应用可能无法被进程环回捕获，表现为勾选后仍无声音；可改用「不过滤」或排除该应用。
- **多应用同时收录**：勾选多个应用时，会为每个目标建立采集流并混音，应用数量过多可能增加 CPU 与延迟，建议仅勾选需要翻译的 1～2 个应用。

## 性能影响

- 设备环回：单路采集，资源占用与当前一致。
- 按应用捕获（外部工具可用时）：每个目标进程一路采集，混音后输出，CPU 略增；若工具不可用则回退为设备环回，无额外开销。

## 验证与排查

1. **确认回退行为**：在过滤器中选择「仅收录所选应用」并勾选一个应用，保存后启动识别与翻译。查看日志：若出现「按应用捕获: 当前系统不支持进程环回，回退为设备环回」或「未找到 AppAudioCapture.exe」，则当前为设备环回（收录全部系统声音），属预期回退。
2. **确认设备环回正常**：选择「不过滤」后启动识别，应能正常识别系统声音；若此处也无声音，请检查默认播放设备与 WASAPI 驱动。
3. **测试捕获按钮**：在过滤器页点击「测试捕获」，可短时录制目标应用音频并查看是否成功捕获，便于判断是否为进程环回或应用兼容问题。
